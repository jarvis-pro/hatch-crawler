# Getting Started

hatch-crawler 的入门资料。**建议阅读顺序**：先看架构总览拿到全貌，再看实施路线知道当前进度和待办。读完这两份就有能力下到 `../reference/` 查具体 schema / API / 看板细节。

## 现有资料

| 文档                                 | 适合阶段                                                         | 阅读时长     |
| ------------------------------------ | ---------------------------------------------------------------- | ------------ |
| [architecture.md](./architecture.md) | 第一篇必读：服务拓扑、模块职责、数据流、关键决策与"不做"清单     | 约 30–45 min |
| [roadmap.md](./roadmap.md)           | 第二篇必读：Phase 1-4 拆解、文件清单、当前验收清单、下一阶段提案 | 约 20 min    |

## 维护约定

- `architecture.md` 是**当前已落地**的系统全貌——新模块上线前更新这里，让它始终反映线上真实状态。
- `roadmap.md` 末尾追加新 Phase 时，每个 Phase 至少给出"目标 / 文件清单 / 验收标准"三项，方便回顾时核对是否真的交付完毕。
- 未来阶段的设计**不进这里**，进 `../rfcs/`，落地后才回填 architecture / roadmap 的"当前状态"小节。
