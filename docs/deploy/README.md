# Deploy

hatch-crawler 的部署蓝图。当前只有一种形态——**Docker Compose 本地 / 单机一键起**，所有进程（Next.js Web / 迁移 / Worker）跑在同一个容器里。本目录每份文档描述**一种部署形态**。

## 形态速查

| 形态        | 文档                             | 状态         | 适用场景                                                    | 主要组件                                    |
| ----------- | -------------------------------- | ------------ | ----------------------------------------------------------- | ------------------------------------------- |
| 本地 / 单机 | [deployment.md](./deployment.md) | **生产可用** | 个人 / 小团队、把数据留在自己机器上、无需高可用             | Docker Compose · Postgres · 单 Next.js 容器 |
| 多节点      | —                                | **未规划**   | 有水平扩展需求时新增（参考 RFC 流程，先在 `../rfcs/` 立项） | —                                           |

## 写新部署形态文档时必须包含的章节

为方便横向对照，每份部署文档保留以下章节顺序：

1. **状态** — 生产可用 / 试验 / 占位符（写在文档开头的引用块里）
2. **拓扑结构** — ASCII 图，覆盖入口、Web、DB、对象存储（如有）、定时任务驱动
3. **关键环境变量** — 至少包含 `DATABASE_URL`、对象存储凭证（如有）、Worker 并发参数
4. **部署步骤** — 从零到可访问看板可重复跑通的命令清单
5. **后台任务驱动** — 当前是 Worker 进程内置 `setInterval`；未来若拆出独立 cron，在这里说明驱动方式
6. **排错指南** — 常见报错 → 定位步骤 → 修复方法
7. **数据备份与回滚** — Postgres dump / 还原命令、迁移回滚策略
